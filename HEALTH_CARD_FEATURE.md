# Health Card Feature - User Guide

## Overview
The new **Health Card** feature provides users with a comprehensive view of their health status based on all uploaded medical reports.

## Features Added

### 1. **Web Dashboard (Frontend)**
**New Component**: `HealthCard.jsx`

**Location in Dashboard**: 
- New "💚 Health Card" tab in the sidebar
- Available after login

**What It Shows**:
- **Overall Health Score** (0-100%)
  - Calculates the percentage of metrics within normal range
  - Color-coded status: Excellent (80%+), Good (60%+), Fair (40%+), Needs Attention (<40%)

- **Metrics Trend Chart**
  - Line chart showing how your metrics change over time (last 5 reports)
  - Visual representation of health trends

- **Metrics Grid**
  - **Values Needing Attention** section showing abnormal metrics
  - Each metric shows:
    - Current value
    - Normal range
    - Status (LOW/HIGH/NORMAL)
    - Color-coded indicators
  - **Normal Values** section showing healthy metrics

- **Recommendations Section**
  - "What You Need To Do" based on all reports
  - Personalized health tips aggregated from all medical reports

### 2. **Mobile App (React Native)**
**New Screen**: `HealthScreen.js`

**Location in Navigation**: 
- New "Health" tab in bottom tab navigator (with ❤️ icon)
- Available in the main app tabs

**What It Shows**:
- Same health score calculation as web
- Metric cards with status indicators
- Scrollable list of metrics and recommendations
- Report information (latest report date, total reports)
- Pull-to-refresh functionality
- Disclaimer about AI analysis

## Supported Health Metrics

The feature tracks over 20 health metrics including:
- **Blood Metrics**: Hemoglobin, WBC, RBC, Platelets
- **Glucose & Metabolism**: Blood Glucose, TSH
- **Heart Health**: Systolic BP, Diastolic BP, Cholesterol, Triglycerides, HDL, LDL
- **Kidney Function**: Creatinine, Blood Urea
- **Liver Function**: SGPT (ALT), SGOT (AST)
- **And more...**

Each metric includes:
- Normal range for that metric
- Status indicator (Low/Normal/High)
- Category (Blood, Heart Health, Kidney, Liver, etc.)
- Icon for easy identification

## How User Health Data Flows

1. **User Uploads Report** → Report is processed by backend
2. **Metrics Extracted** → Health values extracted and stored in database
3. **Recommendations Generated** → AI generates personalized health tips
4. **Health Card Aggregates** → All reports are analyzed together
5. **Overall Score Calculated** → Percentage of normal metrics shown
6. **Trends Visualized** → Charts show health progression over time

## Key Features

✅ **Real-time Health Score** - Updated as users upload new reports
✅ **Trend Visualization** - See how your health is changing over time
✅ **Actionable Recommendations** - Get specific health tips based on your condition
✅ **Anomaly Highlighting** - Abnormal metrics are prominently displayed
✅ **Mobile Optimized** - Works seamlessly on both web and mobile
✅ **Color-Coded Status** - Easy-to-understand visual indicators
✅ **Pull-to-Refresh** - On mobile, swipe down to refresh health data

## Data Sources

- **Metric Data**: Extracted from uploaded medical reports (blood tests, vitals, etc.)
- **Recommendations**: Generated by AI based on metric values and medical knowledge
- **Aggregation**: All user's reports are analyzed together for comprehensive health picture

## User Experience Journey

1. User logs in/registers
2. User can see "Health Card" tab in sidebar/navigation
3. If no reports yet → Shows "No Health Records Yet" message with upload prompt
4. After uploading first report → Health Card becomes populated
5. User can see:
   - Overall health percentage
   - Which metrics are normal and which need attention
   - What actions to take
6. As more reports are uploaded → Trends become clearer, health score updates

## Technical Implementation

**Frontend**:
- React with Hooks for state management
- Framer Motion for animations
- Recharts for data visualization
- Axios for API calls

**Mobile**:
- React Native
- AsyncStorage for caching
- ScrollView with refresh
- Native icons from Ionicons

**Data Flow**:
- Uses existing `/api/reports/` endpoint
- No new backend endpoints needed
- All aggregation happens client-side for performance
- Real-time updates when new reports are uploaded
